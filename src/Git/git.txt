1. Git- система контроля версий, хранилище, база данных истории разработки проекта. Гит нужен для того, чтобы сохранять все версии файлов, которые когда либо были созданы в проекте. Мы добавляем туда стартовую версию файлов, и если что-то изменилось, то добавляем вторую, третью и т.д И можем без проблем откатываться на предыдущие коммиты.
2. Ветки - это перемещаемый указатель на опреденный коммит, Для того, чтобы создать новую ветку нужно написать git branch (name) название новой ветки.
3. merge- тут происходит слияние содержимого веток в одну целевую. при merge меняется только целевая ветка, остальное остается неизменным.
  rebase- тут происходит сжатие всех изменений в одно целое , которое переносится в целевую ветку. в отличии от merge, тут удаляется все что было сжато в одно целое
4. GitHub - сервис, откуда можно брать сохраненные версии файлов проекта. Самые популярные хостинги Гит это github, gitlab, bitbucker.
5. Для того, чтобы клонировать репозиторий, нужно в нем нажать на code, скопировать https линку. потом через гитбаш зайти в место , куда нужно клонировать, ввести git clone и ссылку которую мы копировали.
6. Чтобы отменить последний коммит: git reset.
7. Решение конфликта слияния: pull requests → где запросы на вытягивание нужно нажать на вытягивание вместе с конфликтом слияния → разрешить конфликты.
8. Git reset - инструмент для отмены изменений, который имеет 3 основные формы вызова (-soft, --mixed, --hard)
